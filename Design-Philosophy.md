# Design Philosophy

I didn't start this project because I wanted another Linux rice.

I started it because after installing Linux for the first time on an old 2012 laptop, I realised something felt... missing.

The freedom was incredible. Within a few hours I was changing things I didn't even know were possible. But the deeper I went, the more I noticed there didn't seem to be much of a middle ground.

You either use a polished desktop like GNOME or KDE, where most of the decisions have already been made for you, or you build something yourself with Hyprland. Both are fantastic in their own ways, but one hides the machinery and the other hands you every bolt and tells you to start building.

I don't think those should be the only choices.

I think you should be able to install a desktop that already feels complete, while still being able to open every file, understand every decision, and make it your own.

That's the idea behind this project.

Not to replace GNOME.

Not to replace KDE.

Not even to replace Hyprland.

Just to explore what sits between them.

---

I don't think computers should feel complicated.

Complex? Absolutely.

Powerful? Definitely.

But not complicated.

The more I've learnt about Linux, the more I've realised that complexity isn't the enemy. Hidden complexity is.

When something breaks, I don't want the answer to be "trust the system."

I want to know why.

If I delete a file, I want to know where it went.

If a widget exists, I want to know what it's doing.

If I don't like the way something looks, I want to know exactly where to change it.

That's the kind of desktop I want to build.

---

There are a few ideas that I keep coming back to.

Light.

Space.

Movement.

Buildings.

Water.

Trees.

At first they don't seem to have much in common, but I think they all point towards the same thing.

Good design shouldn't get in the way.

Water doesn't force itself through a mountain. It slowly finds another path.

Great cathedrals weren't impressive because they were enormous. They were impressive because they made light feel like part of the architecture.

Trees don't stop growing because someone adds another branch.

I think software should behave the same way.

It should grow.

It should adapt.

It should let people build on top of it without asking permission.

---

Every feature should have a reason to exist.

Not because it looks cool.

Not because another desktop has it.

Because someone will genuinely benefit from it.

If I ever find myself adding something just because I can, then I've probably started building the wrong project.

---

I don't know exactly what this project will become yet.

That's part of the reason I'm building it in public.

Maybe it'll stay a Hyprland shell.

Maybe it'll eventually become something much bigger.

Either way, I want every decision to be documented, every file to be understandable, and every part of the system to be open to the people using it.

That's the desktop I wish I'd found when I first installed Linux.

So I decided to start building it.

I'm calling it **Project Churchill**.
