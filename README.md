# Project Churchill

I decided to build Project Churchill after running Linux for the first time on an old 2012 laptop.

I expected something lightweight and practical. What I didn't expect was how much freedom Linux gave me. Within a few hours I had gone from installing a new operating system to asking bigger questions about how a desktop should actually work. That eventually led me to Hyprland, AGS, and, after a lot of experimenting, Churchill.

Project Churchill started because I felt there was a gap.

If you want a polished desktop today, GNOME and KDE are fantastic projects with decades of work behind them. If you want complete freedom, Hyprland gives you exactly that. But getting from a fresh installation to something that feels cohesive, beautiful, and personal can take weeks of configuration, reading documentation, and piecing together other people's dotfiles.

I wanted to see if there was another path.

Churchill is my attempt to build a desktop shell that feels intentional from the moment you install it. I want it to be lightweight without feeling unfinished, beautiful without becoming distracting, and powerful without making beginners feel like they're constantly fighting the system.

## What Churchill stands for

### Accessibility

A polished Hyprland setup shouldn't require months of work. Churchill aims to provide an experience that looks and feels complete while still remaining fast and lightweight.

### Freedom

Nothing is hidden.

Every configuration file, script, and component is available, documented, and designed to be understood. If you want to change something, you should be able to.

### Learning

Churchill isn't just something to install. I want it to be something people can learn from. My goal is that someone completely new to Linux can explore the project, understand how it works, and slowly make it their own.

### Intentional design

Every feature should earn its place.

I'm not interested in adding effects simply because they look impressive. Every animation, widget, shadow, and setting should improve the experience in some way.

## Current status

Project Churchill is still in its early stages.

Right now I'm building the foundations: the shell, the theme engine, the settings system, and the design language that will tie everything together.

This repository is also my development journal. I'll be documenting the good ideas, the bad ones, and everything I learn along the way. Hopefully, someone else can learn something from it too.
